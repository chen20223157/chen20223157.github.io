gitLaB问题

Unable to negotiate with 10.10.72.251 port 22: no matching host key type found. Their offer: ssh-rsa,ssh-dss
Could not read from remote repository.

Please make sure you have the correct access rights
and the repository exists.

复制这个到git的etc\ssh\ssh.config中

Host *
HostkeyAlgorithms +ssh-rsa
PubkeyAcceptedKeyTypes +ssh-rsa

如果已经存在一个github的

1. 生成 GitLab 专属 SSH 密钥（指定密钥名称，避免与 GitHub 密钥冲突）

ssh-keygen -t rsa -C “你的GitLab注册邮箱” -f ~/.ssh/id_rsa_gitlab_xunlei

2. 查看 GitLab 专属公钥（后续添加到 GitLab 账号）

cat ~/.ssh/id_rsa_gitlab_（公司名）.pub

3. 编辑 SSH config，指定 GitLab 服务器使用专属密钥

vi ~/.ssh/config
在 config 文件中添加：

匹配 GitLab 企业服务器

Host gitlab.（公司域名）.cn
HostName gitlab.(公司域名).cn
HostKeyAlgorithms +ssh-rsa
PubkeyAcceptedAlgorithms +ssh-rsa
IdentityFile ~/.ssh/id_rsa_gitlab_（公司名） # 指向 GitLab 专属私钥
User git
GSSAPIAuthentication no
PasswordAuthentication no
