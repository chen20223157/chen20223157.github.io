有关我的第一个开源项目 —— 谷粒学院在线教育平台

作为一名刚接触企业级开发的开发者，我一直想通过一个完整项目打通前后端技术链路，而在线教育行业的爆发式增长让我看到了需求痛点 —— 传统教育资源分配不均、学习时间碎片化。于是，我基于 Spring Cloud 微服务架构开发了谷粒学院在线教育平台，并决定将其开源，希望能为同样处于学习阶段的开发者提供参考。以下是项目的完整开发历程、核心实现与心路记录。

一、项目概览：从需求到技术栈

1.1 项目定位与核心功能

谷粒学院是一个面向多角色（学员、讲师、管理员）的 B2C 模式在线教育平台，覆盖10 个教育细分领域（母婴、学前教育、少儿外语等），支持课程管理、讲师管理、视频点播、在线支付等核心功能，解决 “随时随地学习” 和 “教育资源共享” 两大核心需求。

1.2 技术栈选型（附选型理由）

分类 技术选型 选择理由
后端框架 Spring Boot + Spring Cloud 微服务架构支持高可用、可扩展，适合项目后期横向扩展（如拆分课程、用户、支付服务）
持久层 MyBatis-Plus 简化 CRUD 代码，内置分页、逻辑删除、乐观锁，减少重复开发
前端框架 Vue 2 + Element UI + Nuxt.js Vue 生态成熟，Element UI 快速搭建后台，Nuxt.js 解决前端 SEO 问题
中间件 Nacos + RabbitMQ + Redis Nacos 做服务注册 / 配置中心，RabbitMQ 解耦服务，Redis 缓存热点数据（如课程列表）
云服务 阿里云 OSS + 视频点播 降低自建存储 / 视频服务的成本，支持大文件上传和视频转码
构建工具 Maven + Docker Maven 统一依赖管理，Docker 实现环境一致性，便于开源后他人部署

四、问题与解决方案：踩过的坑

问题现象 原因分析 解决方案
前端调用后端接口报 “Access-Control-Allow-Origin” 浏览器同源策略限制（前端 8080，后端 8001） 1. 后端 Controller 加@CrossOrigin注解；2. 生产环境用 Nginx 反向代理统一端口
Feign 调用微服务超时（服务间通信失败） Hystrix 默认超时时间 1 秒，视频上传耗时超过 1 秒 配置文件增加：hystrix.command.default.execution.isolation.thread.timeoutInMilliseconds=6000
阿里云 OSS 上传视频报 413（Request Entity Too Large） Nginx 默认限制上传文件大小 1M Nginx 配置client_max_body_size 1024m;，后端配置spring.servlet.multipart.max-file-size=1024MB
课程列表分页查询慢（数据量 10 万 +） 未给查询字段加索引 给edu_course表的title、teacher_id、gmt_create字段添加联合索引

五、心路历程：从 “迷茫” 到 “开源分享”

5.1 初期：技术栈太多，无从下手

刚开始接触微服务时，Nacos、Feign、Hystrix 这些组件让我眼花缭乱，甚至不知道先搭建注册中心还是先写业务代码。后来我参考了 Spring Cloud 官方文档，决定 “从最小可用版本开始”—— 先搭建一个能跑通的单体服务（讲师管理），再逐步拆分为微服务。这个过程让我明白：复杂项目的核心是 “拆解”，而不是一步到位。

5.2 中期：遇到 Bug 崩溃，解决后成就感爆棚

最崩溃的一次是阿里云 OSS 视频上传功能，前后卡了 3 天：先是配置错误导致签名失败，然后是 Nginx 上传大小限制，最后是前端预览地址拼接错误。当终于在页面上看到上传的视频能正常播放时，我甚至截图发了朋友圈 —— 那种 “从 0 到 1 解决问题” 的成就感，是看教程学不到的。

5.3 开源前：担心代码不规范，不敢分享

准备开源时，我反复检查代码：有没有冗余注释？命名是否统一？文档是否清晰？甚至用 ESLint 和 CheckStyle 统一格式。但真正上传 GitHub 后，我发现开源的意义不仅是 “分享”，更是 “交流”—— 有开发者提 Issue 指出 “视频删除时未同步删除 OSS 文件”，我据此优化了代码，这让项目变得更完善。

5.4 现在：收到第一个 Star，坚定开源信心

当看到 GitHub 仓库收到第一个 Star 时，我突然意识到：即使是 “新手项目”，也能帮到同样处于学习阶段的人。现在我会定期更新文档，回复 Issue，甚至计划添加 “在线支付” 和 “直播” 功能 —— 开源让项目有了 “生命力”，也让我的开发之路有了更多动力。

六、总结与未来计划

6.1 项目收获

  1. 技术能力：打通了 “后端微服务 + 前端 Vue + 云服务” 的全链路，掌握了 MyBatis-Plus、Nacos、OSS 等企业级技术的实战用法；
  2. 工程思维：学会了需求分析、技术选型、模块化开发，不再是 “只会写 CRUD 的开发者”；
  3. 开源意识：理解了 “规范代码” 和 “清晰文档” 的重要性，也体会到开源社区的互助精神。
